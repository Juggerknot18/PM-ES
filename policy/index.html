Policy

Public content policy

This page presents the public website governance source in styled HTML. The Markdown file remains the repository source for maintainers.

Purpose

This repository hosts the public website for PM-ES. Its purpose is to explain the project, its high-level architecture, its development status and its collaboration interests without exposing private engineering material.

Public content versus internal documentation

Public content is written for discovery, review and collaboration. Internal engineering documentation may include implementation details, unpublished procedures, private measurements, credentials, local paths or personal data. Internal material is not mirrored here.

Disclosure levels

PM-ES public disclosure levels
LevelMeaning
P0 publicApproved public facts such as mission, domains, high-level architecture and contact route.
P1 public-with-cautionTechnical summaries that avoid exact thresholds, pin maps, private protocols and unreleased evidence.
P2 internalImplementation details, procedures, logs, calibration data and detailed engineering notes.
P3 restrictedSecrets, credentials, private addresses, personal data and sensitive repository information.
Initial release scope. The public website includes P0 and a minimal subset of P1.

Rules

  • No personal data is published.
  • No measured claim is made without public, reproducible evidence.
  • No private-repository mirroring is allowed.
  • Images, schematics, datasets and documents require review before publication.
  • Publications must identify status, version, date, licence and persistent identifier when those exist.

Removal procedure

If information is accidentally published, stop deployment, remove the content, review repository history exposure, disable GitHub Pages if needed, and document the corrective action before restoring publication.
